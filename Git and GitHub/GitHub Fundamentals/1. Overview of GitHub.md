# Introduction to GitHub

## Overview

**GitHub** is a cloud-based platform that **hosts Git repositories and adds collaboration, automation, and project management features on top of Git**.  

While Git is the underlying **version control system**, GitHub is the **service** that enables developers and organizations to store code remotely, collaborate in teams, review changes, integrate CI/CD pipelines, and build open-source ecosystems.

GitHub has become the industry standard platform for **code hosting, collaboration, and developer workflow automation**.

![GitHub Logo](../images/git_and_github.png)

---

## Why We Need GitHub

Git alone works locally — it manages versions on your machine.  
However, modern software development is almost always:

- collaborative
- distributed across teams and locations
- deployed from servers or pipelines
- dependent on backups and remote access

GitHub addresses these needs by providing:

- **a remote central place to store Git repositories**
- **team collaboration tools**
- **automated workflows**
- **visibility for open-source projects**
- **secure backup of codebases**

Without GitHub (or similar platforms), sharing code would require manual file transfers or self-hosted servers.

---

## What GitHub Does for Us

### 1. Remote Repository Hosting

GitHub stores repositories in the cloud.

You can:

- push local repositories
- clone remote repositories
- collaborate from anywhere

Example push flow:

```bash
git remote add origin <remote-url>
git push -u origin main
```

This allows your teammates and CI systems to access the same repository.

---

### 2. Collaboration and Pull Requests

GitHub enables **pull requests (PRs)**, which are used to:

* review code before merging
* discuss design decisions
* enforce quality checks and approvals

Pull requests are central to professional backend workflows.

---

### 3. Issue Tracking and Project Management

GitHub provides:

* Issues
* Labels
* Milestones
* Kanban-style project boards

Teams use these to track:

* bugs
* feature requests
* tasks
* roadmaps

This turns GitHub into both a **code host** and a **lightweight project management tool**.

---

### 4. CI/CD and Automation with GitHub Actions

**GitHub Actions** allows you to:

* run automated tests
* build code
* deploy services
* perform static analysis
* run pipelines on pushes or pull requests

This eliminates the need for separate CI servers in many projects.

---

### 5. Open Source and Developer Portfolio

GitHub is also the backbone of:

* open-source contributions
* public libraries and frameworks
* personal portfolios for developers

Recruiters and companies frequently evaluate GitHub activity.

---

## Git vs GitHub (Quick Reminder)

* **Git** → version control tool (local)
* **GitHub** → cloud platform built around Git (remote)

GitHub uses Git internally but provides features Git alone does not offer, such as:

* pull requests
* issue tracking
* PR reviews
* repository permissions
* hosted CI/CD

---

## Security and Access Control

GitHub enables:

* private repositories
* organization-level controls
* branch protection rules
* required reviews and checks

This is essential for enterprise backend systems handling sensitive code.

---

## Real-World Backend Engineering Use Cases

Backend teams use GitHub for:

* microservice repositories
* API development collaboration
* infrastructure-as-code storage
* deployment through GitOps
* versioning database migration scripts
* storing documentation and design decisions

In short, GitHub becomes the **central hub of development workflow**.

## Interview Questions

### 1. What is GitHub and how is it different from Git?

**Answer:**
GitHub is a **cloud platform that hosts Git repositories and adds collaboration tools**, while Git is the **version control system** that tracks code changes locally. GitHub builds on top of Git, but Git can exist without GitHub.

---

### 2. Why do we need GitHub in software development?

**Answer:**
We need GitHub for **remote backup, collaboration, code review, CI/CD automation, and team management**. It enables distributed teams to work on the same repository from anywhere.

---

### 3. What is a pull request in GitHub?

**Answer:**
A pull request is a collaborative feature where developers propose changes to a branch, review code, discuss modifications, and merge only after approval and checks.

---

### 4. Can Git be used without GitHub?

**Answer:**
Yes. Git works fully offline and without GitHub. GitHub simply provides **remote hosting and collaboration features** on top of Git.

---

### 5. How is GitHub useful for backend engineers specifically?

**Answer:**
Backend engineers use GitHub to manage microservices, APIs, deployment pipelines, infrastructure code, version migrations, and to collaborate through code reviews and pull requests.

---

## Summary

* GitHub is a cloud platform for hosting Git repositories

* It enables collaboration, pull requests, issues, and automation

* Git is the version control engine; GitHub builds features around it

* GitHub is essential for modern team-based backend development

* It powers CI/CD pipelines, code reviews, project tracking, and open-source work
