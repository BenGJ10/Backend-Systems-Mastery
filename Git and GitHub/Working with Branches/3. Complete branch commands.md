# Complete Branch Commands in Git

## Overview

Branches are a core mechanism in Git that enable **parallel, isolated development**.  

To use branches effectively in backend engineering workflows, it is essential to understand how to **create, switch, rename, and delete branches**, and how Git behaves when switching branches with **unstaged or uncommitted changes**.

---

## Creating Branches

### Create a New Branch

```bash
git branch feature-auth
```

Creates a new branch based on the current commit without switching to it.

This is commonly used when preparing branches in advance.

---

### Create and Switch to a Branch

```bash
git checkout -b feature-auth
```

This command:

* Creates a new branch
* Switches the working directory to it

---

## Switching Branches

### Using `git checkout`

```bash
git checkout feature-auth
```

Historically, `checkout` has been used to:

* Switch branches
* Restore files
* Checkout specific commits

This multi-purpose behavior makes it powerful but sometimes confusing.

---

### Using `git switch`

```bash
git switch feature-auth
```

Introduced to make branch switching **clearer and safer**.
It is dedicated only to branch-related operations.

---

### Checkout vs Switch

| Aspect                    | `git checkout`        | `git switch`            |
| ------------------------- | --------------------- | ----------------------- |
| Purpose                   | Multi-purpose command | Branch-specific command |
| Can switch branches       | Yes                   | Yes                     |
| Can restore files         | Yes                   | No                      |
| Recommended for beginners | No                    | Yes                     |
| Introduced in             | Early Git             | Git 2.23                |

Modern workflows prefer `git switch` for clarity.

---

## Renaming Branches

### Rename Current Branch

```bash
git branch -m new-branch-name
```

Renames the branch you are currently on.

---

### Rename a Different Branch

```bash
git branch -m old-name new-name
```

Useful for correcting naming mistakes or standardizing conventions.

---

## Deleting Branches

### Delete a Local Branch (Safe)

```bash
git branch -d feature-auth
```

Deletes the branch only if it has been fully merged.
Prevents accidental data loss.

---

### Force Delete a Branch

```bash
git branch -D feature-auth
```

Deletes the branch regardless of merge status.
Should be used with caution.

---

## Switching Branches with Unstaged Changes

### What Happens When Changes Are Unstaged?

If you attempt to switch branches with **uncommitted changes**, Git checks whether those changes would be overwritten.

#### Case 1: No Conflict

If changes do not conflict:

```bash
git switch main
```

Git allows the switch and carries the changes over.

---

#### Case 2: Conflict Exists

If switching would overwrite changes:

```text
error: Your local changes would be overwritten by checkout
```

Git blocks the operation to prevent data loss.

---

## Handling Unstaged Changes Safely

### Option 1: Commit the Changes

```bash
git add .
git commit -m "WIP: partial implementation"
```

Safest approach for preserving work.

---

### Option 2: Stash the Changes

```bash
git stash
git switch main
```

Temporarily saves changes and cleans the working directory.

Restore later:

```bash
git stash pop
```

---

### Option 3: Discard Changes (Not Recommended)

```bash
git reset --hard
```

Deletes all uncommitted changes permanently.

---

## Listing Branches

```bash
git branch
```

Shows all local branches.

```bash
git branch -a
```

Shows local and remote branches.

## Backend Workflow Perspective

In backend projects:

* Feature branches isolate API changes
* Bug-fix branches protect production stability
* Proper branch cleanup keeps repositories maintainable

Poor branch discipline often leads to unstable deployments and difficult debugging.
