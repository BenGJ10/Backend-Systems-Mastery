# Prerequisite: Basic Terminal Commands

## Overview

Before working with Git or backend development tools, it is essential to be comfortable with **basic terminal commands**.  
The terminal provides direct interaction with the operating system and is the primary interface for running Git commands, build tools, servers, and deployment scripts.

These commands are **foundational** and are assumed knowledge in professional backend workflows.

---

## Understanding the Terminal Environment

The terminal allows you to execute commands inside a **working directory**.  
Most Git operations, build commands, and application executions are scoped to the current directory, making directory navigation a critical skill.

All examples below apply to **Unix-based systems** (Linux, macOS), which are standard in backend development environments.

---

## Directory Navigation Commands

### `pwd` — Print Working Directory

Displays the absolute path of the current directory.

```bash
pwd
```

Used to confirm where you are before running commands that modify files.

---

### `ls` — List Files and Directories

Lists the contents of the current directory.

```bash
ls
```

Common variations:

```bash
ls -l    # detailed view
ls -a    # includes hidden files
```

Helpful for inspecting project structure.

---

### `cd` — Change Directory

Moves between directories.

```bash
cd backend-service
```

Special cases:

```bash
cd ..    # move one level up
cd ~     # move to home directory
```

Incorrect directory context is a common source of command errors.

---

## File and Directory Management

### `mkdir` — Create a Directory

Creates a new directory.

```bash
mkdir logs
```

Used for organizing configuration, scripts, or resources.

---

### `touch` — Create an Empty File

Creates a new empty file.

```bash
touch application.yml
```

Often used for configuration or placeholder files.

---

### `rm` — Remove Files or Directories

Deletes files or directories.

```bash
rm temp.txt
```

For directories:

```bash
rm -r build
```

Use with caution—deleted files do not go to a recycle bin.

---

### `cp` — Copy Files or Directories

Copies files.

```bash
cp application.yml application-dev.yml
```

---

### `mv` — Move or Rename Files

Moves or renames files.

```bash
mv old-config.yml config.yml
```

---

## Viewing File Contents

### `cat` — Display File Content

Prints file content to the terminal.

```bash
cat README.md
```

Best for small files.

---

### `less` — View Large Files Safely

Allows scrolling through large files.

```bash
less server.log
```

Use `q` to exit.

---

## Searching and Filtering

### `grep` — Search Text in Files

Searches for patterns inside files.

```bash
grep "ERROR" application.log
```

Useful for log analysis in backend systems.

---

## Command History and Shortcuts

### `history` — View Command History

Displays previously executed commands.

```bash
history
```

Improves productivity by reusing commands.

---

### `clear` — Clear Terminal Screen

```bash
clear
```

Used to reset the terminal view.

---

## Permissions and Execution

### `chmod` — Change File Permissions

Modifies execution permissions.

```bash
chmod +x deploy.sh
```

Required to run scripts.

3. **How would you inspect a large log file without opening it in an editor?**
   Mention `less` and `grep`.

4. **What command helps you confirm your current working directory?**
   Explain the role of `pwd`.

5. **Why should `rm -r` be used cautiously in backend environments?**
   Discuss irreversible deletion risks.

## Summary

* Terminal commands are a prerequisite for Git and backend workflows

* Directory navigation commands control command context

* File management commands modify project structure

* Viewing and searching tools are essential for debugging and log analysis

* Mastery of these basics improves efficiency and reduces operational errors
