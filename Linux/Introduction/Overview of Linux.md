# Introduction to Linux

## Overview

**Linux** is an open-source, Unix-like operating system that powers a vast portion of modern computing infrastructure, including servers, cloud platforms, containers, and embedded systems.

For backend engineers, Linux is not optional — most production services run on Linux-based servers. 

---

## What Is an Operating System?

An operating system (OS) is system software that:

- manages hardware resources  
- runs applications  
- handles memory and processes  
- controls file systems  
- provides security  

Linux acts as the intermediary between **hardware and software**.

---

## Why Linux Is So Widely Used

Linux dominates server environments due to several advantages:

### Stability
Linux systems can run for months or years without requiring restarts.

### Security
Strong permission models and active community patching improve system safety.

### Performance
Efficient resource management makes Linux ideal for high-load backend services.

### Open Source
Anyone can inspect, modify, and distribute Linux, leading to rapid innovation.

### Cost Efficiency
Most distributions are free, reducing infrastructure costs.

---

## What Is the Linux Kernel?

The **kernel** is the core component of Linux that directly interacts with hardware.

Responsibilities include:

- process scheduling  
- memory management  
- device control  
- filesystem handling  
- networking  

Everything else in the OS builds around the kernel.

---

## Linux Distributions (Distros)

A **distribution** packages the Linux kernel with tools, libraries, and package managers.

Common server distributions:

- Ubuntu Server  
- Debian  
- Red Hat Enterprise Linux (RHEL)  
- Rocky Linux  
- AlmaLinux  

Cloud environments frequently standardize on one distro.

---

## Key Characteristics of Linux

### Multiuser
Multiple users can operate the system simultaneously.

### Multitasking
Linux runs many processes concurrently.

### Portable
Runs on servers, desktops, IoT devices, and supercomputers.

### Highly Customizable
Users can modify nearly every aspect of the system.

---

## Linux vs Other Operating Systems

| Feature | Linux | Windows | macOS |
|--------|---------|------------|------------|
| Source | Open | Proprietary | Proprietary |
| Server Usage | Dominant | Limited | Rare |
| Customization | Very High | Moderate | Limited |
| Cost | Usually Free | Licensed | Licensed |
| Backend Adoption | Extremely High | Moderate | Low |

Most backend infrastructure runs on Linux.

---

## Why Backend Engineers Must Know Linux

Backend systems often require engineers to:

- configure servers  
- monitor processes  
- inspect logs  
- manage networking  
- automate deployments  
- secure machines  

Without Linux knowledge, production troubleshooting becomes difficult.

---

## Where Linux Is Used

Linux powers:

- cloud servers (AWS, GCP, Azure)  
- Docker containers  
- Kubernetes clusters  
- web servers  
- databases  
- networking appliances  

Even Android is Linux-based.

---

## CLI vs GUI

Linux can operate entirely through the **command-line interface (CLI)**.

Why CLI matters:

- faster administration  
- remote server access via SSH  
- script automation  
- lower resource usage  

Backend engineers primarily interact with Linux through the terminal.

---

## Basic Architecture of a Linux System

High-level structure:

```
Applications
↓
System Libraries
↓
Shell
↓
Kernel
↓
Hardware
```

Each layer communicates with the one below it.
